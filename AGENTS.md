# AGENTS.md

## プロジェクトの目的

- Codex、Python、Git、HTML/CSS/JavaScript、AI創作の学習記録をまとめる静的ホームページを育てる。
- 学習過程そのものを発信資産にする。

## 作業範囲

- このプロジェクトフォルダ内だけで作業する。
- フォルダ外のファイルは変更しない。

## 使用してよい技術

- HTML
- CSS
- JavaScript
- Git

## 禁止事項

- npm、外部ライブラリ、React、Next.js、WordPressは使わない。
- 自動投稿、外部サービス連携、ログイン機能、問い合わせフォームは作らない。
- ファイル削除、移動、大量リネームはしない。
- ネットワークアクセスはしない。
- プロジェクト外のファイルを操作しない。

## Codexの作業手順

- 作業前に変更予定ファイルと変更内容を示す。
- ユーザーの承認後に編集する。
- 編集後に `git status --short`、`git diff --stat`、`git diff` を確認する。
- 表示確認を行う。
- 問題がなければ commit する。
- 最後に変更ファイル、確認結果、commit ID を報告する。

## Git運用

- 1つの作業単位ごとに1つのcommitを作る。
- commit message は短く分かりやすくする。
- 作業後は `git status --short` が出力なしになる状態を目標にする。

## 文章方針

- 硬すぎない自然な日本語にする。
- 学習途中の記録として、覚えたこと、つまずいたこと、試したことを残す。
- 大げさな成功談にしない。
- まず小さく作って、壊さず育てる雰囲気を大切にする。

## AGENTS.md の更新ルール

- AGENTS.md は一度作って終わりではなく、プロジェクト方針が変わった時に更新する。
- ただし、毎回の小さな作業内容は書き込まない。
- 更新が必要な時は、先に変更理由と変更予定箇所を提示する。
- ユーザーの承認後に編集する。
- AGENTS.md を更新した場合は、専用のcommitを作る。
