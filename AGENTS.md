# AGENTS.md

## プロジェクトの目的

- Codex、Python、Git、HTML/CSS/JavaScript、AI創作の学習記録をまとめる静的ホームページを育てる。
- 学習過程そのものを発信資産にする。

## 作業範囲

- このプロジェクトフォルダ内だけで作業する。
- フォルダ外のファイルは変更しない。

## 使用してよい技術

- HTML
- CSS
- JavaScript
- Git

## 禁止事項

- npm、外部ライブラリ、React、Next.js、WordPressは使わない。
- 自動投稿、外部サービス連携、ログイン機能、問い合わせフォームは作らない。
- ファイル削除、移動、大量リネームはしない。
- ネットワークアクセスはしない。
- プロジェクト外のファイルを操作しない。

## Codexの作業手順

- 作業前に変更予定ファイルと変更内容を示す。
- ユーザーの承認後に編集する。
- 編集後に `git status --short`、`git diff --stat`、`git diff` を確認する。
- 表示確認を行う。
- 問題がなければ commit する。
- 最後に変更ファイル、確認結果、commit ID を報告する。

## 作業レベル別の運用ルール

### レベル1：軽微な変更

対象：

- README.md の短い修正
- HTML内の文章追記・文言修正
- learning.html の学習メモ追加
- articles.html の説明文調整
- about.html の短い説明文修正
- 既存リンク文言の小変更

運用：

- Codexは変更予定ファイルと変更内容を提示する。
- ユーザー承認後に編集する。
- 編集後は `git status --short`、`git diff --stat`、`git diff` を確認する。
- 表示確認できる場合はローカル表示確認を行う。
- 問題がなければ commit する。
- 必要に応じて push する。
- ChatGPT確認は必須ではない。

### レベル2：中くらいの変更

対象：

- 新しいページ追加
- 複数HTMLファイルの変更
- CSSクラス追加
- 画像1枚の追加・差し替え
- 記事ページの共通導線変更
- 公開URL上の見た目に関わる変更

運用：

- Codexは変更予定ファイル、変更内容、影響範囲、リスクを提示する。
- ユーザーが必要と判断した場合はChatGPTに貼って分析する。
- 承認後のみ編集する。
- 編集後はローカル表示確認、スマホ幅確認、コンソールエラー確認を行う。
- 問題がなければ commit し、必要に応じて push する。
- 公開URLに関わる場合はGitHub Pages上でも確認する。

### レベル3：大きな変更・危険な変更

対象：

- ディレクトリ構成変更
- ファイル削除・大量移動・大量リネーム
- JavaScriptの大きな変更
- CSS全体設計の変更
- GitHub Pages設定変更
- 外部リンクの本番URL化
- npm install
- 外部ライブラリ導入
- React / Next.js / WordPress 導入
- 自動投稿機能
- 外部API連携
- Git履歴操作
- `git reset`、`git checkout --`、force push

運用：

- Codexは編集しない。
- 変更予定、リスク、代替案、Codexとしての推奨を提示する。
- ユーザーがChatGPTに貼って分析する前提で停止する。
- 明確な承認があるまで、編集・commit・pushを行わない。

## 時間短縮用の報告形式

Codexの報告は、原則として以下の形式にまとめる。

- 作業レベル：
- 変更予定ファイル：
- 変更内容：
- リスク：
- 承認が必要な点：
- Codexとしての推奨：
- 次に行う作業：

## 短縮指示の扱い

ユーザーが「軽微な修正として進めて」「レベル1で進めて」「まず予定だけ出して」と指示した場合、CodexはAGENTS.mdの作業レベル別ルールに従って進める。

ただし、実際に確認した結果、レベル2またはレベル3に該当すると判断した場合は、ユーザーの指示より安全判断を優先し、編集前に停止して報告する。

## Git運用

- 1つの作業単位ごとに1つのcommitを作る。
- commit message は短く分かりやすくする。
- 作業後は `git status --short` が出力なしになる状態を目標にする。

## 文章方針

- 硬すぎない自然な日本語にする。
- 学習途中の記録として、覚えたこと、つまずいたこと、試したことを残す。
- 大げさな成功談にしない。
- まず小さく作って、壊さず育てる雰囲気を大切にする。

## AGENTS.md の更新ルール

- AGENTS.md は一度作って終わりではなく、プロジェクト方針が変わった時に更新する。
- ただし、毎回の小さな作業内容は書き込まない。
- 更新が必要な時は、先に変更理由と変更予定箇所を提示する。
- ユーザーの承認後に編集する。
- AGENTS.md を更新した場合は、専用のcommitを作る。
